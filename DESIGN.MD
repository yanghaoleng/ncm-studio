# DESIGN

## 设计定位

- 工具感优先：信息清晰、操作直接，不做营销型首屏。
- 主工作区采用单列布局，NCM 文件处理面板占据可用宽度，空状态保持紧凑。
- 整体使用白/深灰中性底色，蓝色用于主操作，黄色用于等待警告，红色用于错误。
- 标题字体使用 `Avenir Next / Gill Sans / PingFang SC`，正文使用系统无衬线字体。

## 可复用参数

- 面板圆角：14-18px；按钮圆角：10px；紧凑提示圆角：10px。
- 常规交互过渡：180ms ease；状态颜色过渡：260ms ease。
- 按钮高度：最小 42px；图标通常为 16-18px。
- NCM 标题下方的其他平台提示使用 12px、左对齐并允许自动换行；整句作为主色官方链接，末尾配 13px 外链图标。
- 空状态标题区与文件拖入区使用单层 8px 间距，避免标题栏下边距重复叠加。

## ZIP 打包状态

- 打包按钮文字显示实时整数百分比，并用从左向右增长的底色表示进度。
- 打包中按钮保持 100% 不透明，光标为 `wait`，避免误解为不可用。
- 卡顿提示使用 `--yellow-soft`，失败提示使用 `--rose-soft`，均使用 12px 文字和 17px 警告图标。
- 进度条宽度过渡为 180ms ease；遵循 `prefers-reduced-motion`。

## 封面一致性

- 列表缩略图、播放器预览和下载 MP3 使用同一份 NCM 封面数据。
- MP3 封面写入标准 ID3v2.3 `APIC` 前封面帧，ZIP 打包不得替换为未写标签的原始音频。

## CLI 交互

- 人类模式使用简短中文进度：`[当前/总数] 百分比 输入 -> 输出`。
- 机器模式保持 stdout 为单个 JSON 对象，进度只写入 stderr。
- 成功退出码为 0，参数、输入或任何文件转换失败时为 1。

## CLI 安装窗口

- 桌面端工作区使用主面板 + 300px 右侧窗口，间距 14px；920px 以下改为单列。
- 窗口使用 14px 圆角和中性表面，绿色仅用于终端图标和复制成功反馈。
- 安装地址行本身是唯一复制控件，“安装链接”作为框内前缀，只复制 npm 地址。
- 成功后复制图标短暂变为绿色对钩，2.2s 后恢复；不显示额外操作行，不改变布局。
